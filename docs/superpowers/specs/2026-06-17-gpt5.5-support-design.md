# GPT-5.5 対応 設計

- 日付: 2026-06-17
- 対象: `App::Greple::xlate`（gpt5 エンジン）および連携する `gpty` コマンド（`App-gpty`）

## 目的

xlate の gpt5 エンジンが使用するモデルを、OpenAI の最新版 `gpt-5.5` に更新する。
エンジン名 `gpt5` はそのまま維持し、デフォルトモデルだけを `gpt-5` → `gpt-5.5` に差し替える
（利用者は `--xlate-engine=gpt5` のまま最新モデルを使える）。

## 背景・確認した事実（OpenAI 公式）

| 項目 | GPT-5（現状） | GPT-5.5（新） |
|---|---|---|
| モデル ID | `gpt-5` | `gpt-5.5`（snapshot `gpt-5.5-2026-04-23`） |
| 派生 | mini/nano あり | mini/nano なし（無印のみ） |
| 種別 | reasoning model | reasoning model |
| `reasoning_effort` | minimal/low/medium/high | none/low/medium/high/xhigh（`minimal` 廃止・`xhigh` 追加・default medium） |
| `verbosity` | low/medium/high | low/medium/high（low 推奨） |
| temperature | 非対応 | 非対応 |
| context / 出力上限 | 272K / 128K | 1,050,000 / 128,000 |
| 価格 (1M tokens) | in $1.25 / out $10 | in $5.00 / cached $0.50 / out $30.00（>272K 入力で in 2x, out 1.5x） |

出典:
- https://developers.openai.com/api/docs/models/gpt-5.5
- https://developers.openai.com/api/docs/guides/latest-model
- https://platform.openai.com/docs/guides/reasoning

最重要差分: `reasoning_effort` の `minimal` が廃止され、推論を実質オフにする最速設定は `none` になった。
現行 `gpt5.pm` は `minimal` を指定しており、かつ `gpty.py` の検証リストは
`["minimal","low","medium","high"]` のため、`none` を渡しても黙って捨てられる。
→ gpty 側の修正が必須。

## 設計判断

- **エンジン名は `gpt5` のまま**（新モジュールを作らない）。ユーザー指示「gpt5 のデフォルトを 5.5 にしてしまっていい」に従う。
- **`reasoning_effort => 'none'`** を採用。
  - gpt-5 では「デフォルトでは遅すぎた」ため `minimal` を選んだ経緯がある。
  - `none` はその `minimal` の直接の後継（推論を実質オフ、最速・最安）。
  - 翻訳対象は技術文書で、正確さ・分かりやすさが第一・美文は不要。まずは動作と速度優先で `none`。
  - `low`（わずかな推論で曖昧文の取りこぼしを減らせる可能性）との品質比較は**後日、実測で検討**する（別タスク）。
- `verbosity => 'low'`、`max_completion_tokens => 4000` は維持（reasoning=none なので出力枠は十分）。

## 変更内容

### 1. `gpty`（App-gpty / Python: `gpty/gpty.py`）

- `reasoning_effort` の検証リストに `none`, `xhigh` を追加（`minimal` は旧 gpt-5 互換のため残す）。
  - `["minimal","low","medium","high"]` → `["minimal","none","low","medium","high","xhigh"]`
- `aliases` に `"5.5": "gpt-5.5"` を追加（任意・一貫性のため）。
- `temperature` / `max_completion_tokens` / `verbosity` は `engine.startswith("gpt-5")` 分岐で
  `gpt-5.5` も既にカバー済み → 変更不要。
- `Changes` に追記。

### 2. `gpt5.pm`（xlate / Perl: `lib/App/Greple/xlate/gpt5.pm`）

- `%param` の `gpt5` 定義:
  - `engine => 'gpt-5'` → `'gpt-5.5'`
  - `reasoning_effort => 'minimal'` → `'none'`
  - `verbosity => 'low'`, `max_completion_tokens => 4000` は維持
- POD 更新:
  - モデル説明（GPT-5.5）、変種（mini/nano の記述を削除/修正）
  - トークン上限（1,050,000 / 128,000）
  - `reasoning_effort` の値（none/low/medium/high/xhigh、default medium）
  - 価格表（$5.00 / $0.50 cached / $30.00）
  - CONFIGURATION の default 値（engine=gpt-5.5, reasoning_effort=none）

### 3. メインモジュール POD（`lib/App/Greple/xlate.pm`）

- エンジン一覧の `gpt5: gpt-5`（`xlate.pm:195` 付近）を `gpt-5.5` に更新。
- 本文中で gpt5 をモデル `gpt-5` と説明している箇所があれば併せて修正。

## テスト / 検証

- `gpty.py`: 構文・起動確認、`--engine gpt-5.5 --reasoning-effort none --debug` で
  request_params に `reasoning_effort: none` が含まれること（API キー不要、`--debug` で確認）。
- xlate: `perl -c lib/App/Greple/xlate/gpt5.pm`、`./Build test`（既存テスト）。
- 実翻訳の動作確認は OPENAI_API_KEY が必要なため、可能なら代表的な技術文書サンプルで 1 回実行。

## スコープ外（将来）

- `none` vs `low` の翻訳品質の実測比較とデフォルト見直し。
- gpt-5.5 派生（将来 mini/nano が出た場合）の追加対応。
